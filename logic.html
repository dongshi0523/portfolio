“替你选”小程序核心逻辑与架构说明书 V2

一、 产品概述与核心定位

“替你选”是一款旨在帮助用户（主要为00后大学生和上班族）解决“吃什么”决策困难的微信小程序。通过硬性筛选、个性化偏好过滤以及随机加权算法，将繁琐的选择过程转化为带有盲盒惊喜感的抽奖体验，从而大幅缩短用户的决策和思考时间。

二、 核心业务流程 (User Journey)

系统的主干业务主要分为两条抽奖路径，最终汇聚于外卖平台的下单转化：

常规精准抽取路径： 首页 (SCREEN_33) → 点击“开始抽取” → 筛选要求页 (SCREEN_20，设置平台、分类、预算、距离) → 点击开始 → 抽奖过渡动画 (SCREEN_39，大转盘/老虎机) → 最终菜品展示页 (SCREEN_4)。

极简全随机路径（趣味模式）： 首页 (SCREEN_33) → 开启“摇摇”监听 → 摇晃手机触发传感器 → 绕过精细筛选（仅保留营业状态、避雷库、默认5km距离等硬性条件） → 直接进入动画与最终菜品展示。

结果闭环与转化： 在最终展示页，用户可查看高清图、价格和评价。点击“立即下单”则通过微信小程序 API 跳转至对应的第三方外卖平台（美团/淘宝/京东等）；若点击“不满意？再抽一次”，则返回上一层级重新触发算法。

三、 核心引擎：智能抽奖算法逻辑

抽奖算法逻辑旨在通过“硬性筛选+偏好过滤+条件放宽+随机加权”四个维度，实现减少用户决策压力并提供惊喜感的目标。

四、 辅助支撑与生态构建流程

除了核心抽奖工具属性，系统还设计了内容生态和个人管理模块以增加用户粘性：

社区互动（种草与避雷）： 发现页 (SCREEN_10) 允许用户浏览全站的“种草”和“踩雷”卡片，查看详情并通过“去试试”反向引流至抽奖或商品页。配套的发布编辑器（待开发预留）支持上传图片、评价，构建UGC生态。

个人中心与反向控制 (SCREEN_11)： 个人中心不仅用于查看足迹和反馈建议，最核心的逻辑在于“个性化偏好设置”。用户在此处维护的“避雷库”作为底层数据支撑，反向且强制地作用于核心业务流程中的抽奖算法，形成产品的核心壁垒。

五、 潜在技术风险与对策

根据架构评估，该逻辑落地的主要挑战在于第三方交互：

第三方平台跳转： 终极页需跳转美团等平台，获取实时的菜品URL是难点。若无深度API，可能需要回退至跳转小程序首页或搜索结果页。建议初期通过CPS接口获取数据。

性能与LBS精度： “避雷库”规则增多可能导致算法延迟，建议使用 Redis 缓存热门区域数据；同时需利用微信原生高精度定位接口以保证“距离筛选”的有效性。
